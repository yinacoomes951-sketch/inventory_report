# HTML 库存诊断报告指标口径说明

## 1. 文档目的

本文用于帮助业务人员理解 HTML 库存诊断报告中的指标、判断规则和使用边界。

报告主要用于回答以下问题：

- 当前库存整体是否健康；
- 哪些 SPU 存在备货不足、备货过量、发货不足或发货过量；
- 哪些 SPU 存在无动销或长库龄风险；
- 业务应优先补货、发货、控货、清理库存，还是先核查数据。

报告以 SPU 为主要诊断粒度。SKU 主要作为问题证据和 BI 下钻明细，不建议直接根据单个 SKU 的状态判断整个 SPU。

## 2. 数据范围

### 2.1 数据批次

报告使用库存数据源最新 `insert_time` 批次的数据。

### 2.2 权限范围

| 报告对象 | 数据范围 |
| --- | --- |
| 运营个人 | 按“归属”过滤 |
| 战队或部门 | 按“部门名称”过滤 |
| 运营总监 | 全量数据或已授权的组织范围 |

### 2.3 纳入报告的 SPU

数据先按权限过滤，再按 SPU 汇总。

- SPU 汇总预测日销小于 `0.5` 时，报告按 `0` 处理；
- SPU 汇总预测日销等于 `0.5` 时，保留原值；
- 仅当 SPU 的总库存和处理后的预测日销同时为 `0` 时，该 SPU 不纳入报告；
- 库存不为 `0`、但预测日销为 `0` 的 SPU 仍会保留，用于识别无动销风险。

## 3. 核心指标

### 3.1 SPU 数

| 项目 | 说明 |
| --- | --- |
| 指标含义 | 当前报告数据范围内纳入诊断的不同 SPU 数量 |
| 计算方式 | 对 SPU 去重计数 |
| 统计粒度 | 报告范围 |
| 注意事项 | 一个 SPU 下可以包含多个 SKU，但只计为一个 SPU |

### 3.2 整体库存口径

| 项目 | 说明 |
| --- | --- |
| 页面名称 | 整体库存口径 |
| 指标含义 | 当前报告范围内源表“总库存”字段的合计 |
| 计算方式 | `整体库存口径 = Σ 总库存` |
| 单位 | 件 |
| 业务用途 | 与预测日销共同计算整体可售天数，辅助判断是否需要采购或控制备货 |

“总库存”是数据源提供的综合库存字段。报告直接使用该字段，不在报告程序中重新将 FBA、AWD、国内库存、采购在途和采购计划相加。

### 3.3 海外在途及可售

| 项目 | 说明 |
| --- | --- |
| 页面名称 | 海外在途+可售 |
| 数据字段 | 国外合计 |
| 计算方式 | `海外在途及可售 = Σ 国外合计` |
| 单位 | 件 |
| 业务用途 | 用于判断海外供给和发货节奏 |

该指标不包含国内库存、采购在途和采购计划。

### 3.4 预测日销

| 项目 | 说明 |
| --- | --- |
| 指标含义 | 预计每日销售数量 |
| 计算方式 | 先按 SPU 汇总源表“预测日销” |
| 单位 | 件/天 |
| 特殊处理 | SPU 汇总值严格小于 `0.5` 时按 `0` 处理 |
| 业务用途 | 作为整体可售天数和在途+可售天数的统一分母 |

预测日销是覆盖天数的关键分母。若预测明显偏高或偏低，覆盖天数和后续诊断也会随之变化，因此异常结果应优先复核预测。

### 3.5 最近 30 天销量

| 项目 | 说明 |
| --- | --- |
| 页面名称 | 近30天销量 |
| 计算方式 | `Σ 最近30天总销量` |
| 单位 | 件 |
| 业务用途 | 辅助判断近期是否有实际动销，以及长库龄库存是否仍在消化 |

最近 30 天销量用于辅助诊断，不作为覆盖天数的分母。

## 4. 覆盖天数指标

### 4.1 整体可售天数

| 项目 | 说明 |
| --- | --- |
| 指标含义 | 当前全部库存按照预测日销预计可支持销售的天数 |
| 计算公式 | `整体可售天数 = 整体库存口径 ÷ 预测日销` |
| 健康参考区间 | `90–150天`，包含 90 天和 150 天 |
| 低于 90 天 | 备货不足，需复核采购在途、采购计划和预测日销 |
| 高于 150 天 | 备货过量，需控制新增采购并关注动销和库龄 |

整体可售天数用于采购和备货判断，不等同于海外发货可支撑天数。

### 4.2 在途+可售天数

| 项目 | 说明 |
| --- | --- |
| 指标含义 | 海外在途及可售库存按照预测日销预计可支持销售的天数 |
| 计算公式 | `在途+可售天数 = 海外在途及可售 ÷ 预测日销` |
| 健康参考区间 | `60–100天`，包含 60 天和 100 天 |
| 低于 60 天 | 发货不足，优先复核发货或调拨 |
| 高于 100 天 | 发货过量，优先控发并消化海外库存 |

该指标只用于判断海外覆盖和发货节奏，不包含国内库存、采购在途和采购计划。

### 4.3 覆盖天数为空

当处理后的预测日销为 `0` 时，覆盖天数无法计算，报告显示为 `-`，而不是 `0天`。

业务上应将其理解为“当前没有有效日销分母”，需结合库存和近 30 天销量判断是否为无动销、停售、新品冷启动或数据异常。

### 4.4 汇总值与单个 SPU 值

- 报告总览覆盖天数：报告范围内库存合计除以预测日销合计；
- 核心影响 SPU 表中的覆盖天数：单个 SPU 的库存合计除以该 SPU 的预测日销合计。

总览值正常不代表每个 SPU 都正常，应结合各类问题 SPU 数量和明细一起查看。

## 5. 问题 SPU 指标

同一个 SPU 可以同时进入多个问题分类。例如整体库存不足，同时海外覆盖也不足。

| 页面指标 | 判断口径 | 业务含义 |
| --- | --- | --- |
| 备货不足 SPU | 整体可售天数 `< 90天` | 全部库存覆盖不足，需进入补货复核 |
| 备货过量 SPU | 整体可售天数 `> 150天` | 全部库存覆盖偏高，需控制新增备货 |
| 发货不足 SPU | 在途+可售天数 `< 60天` | 海外供给不足，需优先考虑发货或调拨 |
| 发货过量 SPU | 在途+可售天数 `> 100天` | 海外供给偏高，需控制继续发货 |
| 无动销 SPU | 有库存，且 SPU 层近30天销量不大于 0、处理后预测日销为 0 | 有库存但没有有效动销，需人工排查原因 |
| 健康 SPU | 未触发以上任何问题分类 | 当前未触发报告设置的主要问题规则 |

“健康 SPU”是问题分布中的互斥剩余项；其他问题分类之间不互斥，因此各问题 SPU 数相加可能大于 SPU 总数。

## 6. 库龄指标

库龄类指标均为库存数量，单位为件。

### 6.1 国内 90 天以上库龄

```text
国内90天以上库龄
= 国内90–180天
+ 国内180–270天
+ 国内270–330天
+ 国内330–365天
+ 国内365天以上
```

### 6.2 海外 90 天以上库龄

```text
海外90天以上库龄
= 海外3–6个月
+ 海外6–9个月
+ 海外9–11个月
+ 海外11–12个月
+ 海外12个月以上
```

海外数据使用按月划分的库龄区间，因此报告将“3–6个月”及以上作为海外 90 天以上库存。

### 6.3 90 天以上库龄

```text
90天以上库龄 = 国内90天以上库龄 + 海外90天以上库龄
```

该指标表示需要重点观察的长库龄库存总量。

### 6.4 12 个月以上库龄

```text
12个月以上库龄 = 国内365天以上库龄 + 海外12个月以上库龄
```

12 个月以上库存属于严重长库龄风险，应优先评估清仓、移除、停售复核或坏账风险。

## 7. SPU 健康状态

每个 SPU 只有一个主健康状态。判断按以下顺序执行，先命中的状态生效。

| 优先级 | 状态 | 判断口径 |
| --- | --- | --- |
| 1 | 呆滞风险 | 预测日销为 0，且总库存大于 0 |
| 2 | 不健康 | 整体可售天数低于 90 天，或在途+可售天数低于 60 天 |
| 3 | 局部预警 | 整体可售天数高于 150 天，或在途+可售天数高于 100 天，或存在 90 天以上库龄 |
| 4 | 健康 | 未命中以上规则 |

### 7.1 报告整体健康状态

报告整体状态由 SPU 状态分布汇总得出：

1. 呆滞风险 SPU 达到 3 个及以上：整体显示“呆滞风险突出”；
2. 否则，只要存在不健康 SPU：整体显示“不健康”；
3. 否则，只要存在局部预警 SPU：整体显示“局部预警”；
4. 其余情况显示“整体健康”。

整体状态用于快速提示风险，不代表所有 SPU 都处于同一状态。

## 8. 综合影响分

综合影响分用于核心影响 SPU 的排序，分数越高，表示该 SPU 相对更值得优先排查。

```text
综合影响分
= 整体可售天数偏离得分
+ 在途+可售天数偏离得分
+ 呆滞附加分
+ 90天以上库龄附加分
```

具体规则如下：

| 风险项 | 计分方式 |
| --- | --- |
| 整体可售天数低于 90 天 | `(90 - 实际天数) × 8` |
| 整体可售天数高于 150 天 | `(实际天数 - 150) × 3` |
| 在途+可售天数低于 60 天 | `(60 - 实际天数) × 7` |
| 在途+可售天数高于 100 天 | `(实际天数 - 100) × 3` |
| 预测日销为 0 且仍有库存 | 加 160 分 |
| 90 天以上库龄 | `90天以上库龄 ÷ 100`，最高加 80 分 |

综合影响分没有固定满分，不是健康百分比、风险概率或金额，也不能跨不同版本规则直接比较。业务人员应使用它确定排查先后顺序，不能仅凭分数直接决定采购、发货或清仓。

## 9. 核心影响 SPU 表

### 9.1 表内通用指标

| 指标 | 说明 |
| --- | --- |
| 健康状态 | 该 SPU 按第 7 节规则得到的主状态 |
| 本组关注点 | 说明该 SPU 为什么进入当前问题分类 |
| 整体可售天数 | 该 SPU 的总库存除以预测日销 |
| 在途+可售天数 | 该 SPU 的海外在途及可售除以预测日销 |
| 综合影响分 | 用于问题排查排序的规则分 |

### 9.2 不同问题组的补充指标

| 问题组 | 补充展示指标 | 用途 |
| --- | --- | --- |
| 备货不足 SPU | 预测日销 | 判断库存消耗速度和补货紧迫度 |
| 发货不足 SPU | 预测日销 | 判断海外库存消耗速度和发货紧迫度 |
| 备货过量 SPU | 90天以上库龄 | 判断库存过量是否已形成长库龄 |
| 发货过量 SPU | 90天以上库龄 | 判断海外覆盖偏高是否伴随长库龄 |
| 无动销或呆滞 SPU | 90天以上库龄 | 判断是否需进入清仓或坏账风险处理 |

各问题组默认最多展示 8 个优先对象。表格用于定位重点，不等于该问题类型的完整 SPU 清单。

## 10. 业务判断建议

### 10.1 海外不足、整体充足

优先考虑从国内发货或调拨，不应仅因海外不足就直接新增采购。

### 10.2 海外不足、整体也不足

同时复核预测日销、采购在途和采购计划，再决定是否补采购。

### 10.3 海外过量

优先控制继续发货，并检查海外动销和库龄。

### 10.4 整体过量

优先控制新增采购，并检查全部库存位置、动销和库龄。

### 10.5 无动销或长库龄

库存数据只能证明“没有有效动销且仍有库存”，不能直接证明原因。应进一步核查：

- 商品链接和开售状态；
- 是否停售、断货或下架；
- 是否为新品冷启动；
- 预测日销是否异常；
- 库存是否处于不可售位置；
- 是否需要清仓、移除或计提坏账。

## 11. 使用限制

- 报告阈值是当前业务诊断参考线，不等同于财务核算标准；
- “备货预警”等源表标签仅作为辅助信息，最终诊断以 SPU 汇总指标为主；
- 报告不会证明库存异常的具体业务原因，不能确认的原因必须人工核查；
- 总览为聚合数据，不能替代 SPU 分组和 SKU 明细；
- 采购、发货、清仓等最终动作仍需结合交期、季节性、活动计划、商品状态和业务经验确认。
