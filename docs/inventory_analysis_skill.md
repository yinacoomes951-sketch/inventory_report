# 库存诊断分析 Skill

## 目标

把角色范围内的库存明细转换为可审计的结构化诊断。先计算事实，再形成推断；最终报告写作由 `inventory_report_skill.md` 负责。

## 输入范围

- 运营个人：按 `归属` 过滤。
- 战队/部门：按 `部门名称` 过滤。
- 运营总监：全量或授权组织范围。
- 数据批次：使用 `lx_ads.ads_lx_kd_inventory_sku_calc` 最新 `insert_time`。

## 核心原则

1. `备货预警` 仅作参考标签，不能代替完整诊断。
2. 日销统一使用 `预测日销`。
3. 先按 SPU 诊断，再保留 SKU 作为证据和 BI 下钻。
4. 规则和 Python/SQL 负责数字，LLM 不得重新计算或虚构事实。
5. 同一 SPU 可以有多个问题标签，但主健康状态只能有一个。

## 覆盖天数

### 整体库存覆盖

用于判断是否需要进入采购/备货决策：

```text
整体库存 = 总库存字段
整体库存覆盖天数 = 整体库存 / 预测日销
健康参考区间 = 90 至 150 天
```

### 海外覆盖

用于判断发货节奏：

```text
海外覆盖库存 = 海外在途及可售
海外覆盖天数 = 海外覆盖库存 / 预测日销
健康参考区间 = 60 至 100 天
```

判断顺序：

1. 海外不足但整体充足：优先发货或调拨，不建议采购。
2. 海外不足且整体不足：进入采购补货复核。
3. 海外过量：控制继续发货，检查海外库龄。
4. 整体过量：控制新增采购，检查动销和全部库龄。

## 健康状态

- `healthy`：两个覆盖区间合理，持续动销，无明显长库龄。
- `local_warning`：整体可控，但存在局部供给、过量或库龄问题。
- `unhealthy`：关键动销 SPU 的供给或库存结构明显偏离目标。
- `stagnant`：SPU 无动销或极弱动销，同时仍有库存或长库龄。

无动销必须在 SPU 汇总层判断。单个 SKU 无销量不能直接把整个 SPU 判为无动销。

## 库龄规则

- 国内与海外 90 天以上库存进入重点观察。
- 12 个月以上库存单列严重风险。
- 有动销 SPU 的长库龄：局部清理风险。
- 无动销 SPU 的长库龄：清仓、移除或坏账风险。

## 聚合展示

角色总览不得只依赖“总库存 / 总预测日销”。至少同时输出：

- SPU 状态分布；
- 覆盖不足和覆盖过量的 SPU 数；
- 覆盖天数中位数及合理分位数；
- 高销量或高库存核心影响 SPU；
- 90 天以上和 12 个月以上库龄数量。

## 输出要求

结构化 diagnosis 至少包含：

- scope 与 batch；
- summary；
- SPU 健康分布；
- 每类问题的 SPU 列表；
- facts、inference、action、confidence；
- BI 下钻所需的筛选参数；
- 仅供证据使用的 SKU 列表。

任何无法由字段证明的原因必须标为“需人工确认”。
